# Real TCG Card Backgrounds Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace HoloCard's CSS-only background with real 1999 Pokémon TCG card photos (Base Set/Jungle/Fossil, Mew from Wizards Black Star Promos) so cards look 1:1 like https://poke-holo.simey.me/, with our existing holo/foil/cosmos effect layers unchanged on top and our real PokeAPI stats in a footer strip.

**Architecture:** A static `lib/tcg-cards.ts` map (Pokédex # → `images.pokemontcg.io` URL) is added — already generated and validated against the live Pokémon TCG API (151/151 entries, no gaps, conflicts resolved by set priority `base1 > base2 > base3 > basep`). `HoloCard.tsx` gains an optional `tcgImageUrl` prop: when present, the card renders the real card photo full-bleed (`object-fit: contain`) instead of the current sprite-frame layout, with an `onError` fallback to the old layout, plus a new compact stats+button footer below the image. The three callers (`HomePokedexPreview.tsx`, `PokemonCard.tsx`, `PokedexGrid.tsx`) pass `tcgImageUrl={TCG_CARD_IMAGE[id]}`.

**Tech Stack:** React 19, Next.js 16 (App Router), TypeScript, Tailwind CSS v4, existing HoloCard.css effect layers (unmodified)

## Global Constraints

- TypeScript strict mode — no `any`, no `@ts-ignore`
- `npm run lint` must pass (0 errors; 1 pre-existing warning in layout.tsx is allowed)
- `npx vitest run` must show 34 passing tests (existing suite; no new tests required for this pure UI/data change)
- `npm run build` must succeed
- No new npm dependencies
- All text stays English
- Dev server runs on port 3001 (`npm run dev`)
- The `lib/tcg-cards.ts` mapping is static and committed — no runtime fetch to `api.pokemontcg.io`. Only the CDN image hotlink (`images.pokemontcg.io/...`) is used at runtime, as a plain `<img src>`.
