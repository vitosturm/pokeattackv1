# Spec: Real TCG Card Backgrounds for HoloCard

Date: 2026-06-23

## Goal

Replace HoloCard's CSS-only background with real, photographed Pokémon TCG card
images (1999 Base Set / Jungle / Fossil) pulled from the official Pokémon TCG
API's image CDN (`images.pokemontcg.io`), so the cards look 1:1 like
https://poke-holo.simey.me/ — while keeping our holo/foil/cosmos effect layers
(already implemented) unchanged on top, and keeping our own PokeAPI stats
visible in a footer strip.

## Background

- poke-holo.simey.me composites its holo effects over a **real TCG card photo**
  loaded from `images.pokemontcg.io/{setId}/{number}_hires.png` (Pokémon TCG
  API, a separate, free, key-optional API — not PokeAPI).
- Our holo/foil/glare/cosmos CSS layers (`HoloCard.css`) already replicate
  Simey's effect technique exactly. The only remaining gap is the **base card
  image** itself: we currently render our own simplified frame (name/HP header,
  sprite box, type badge, stat grid) instead of a real card photo.
- pokemontcg.io is still operational (free, 1000 req/day without a key) but
  development focus has shifted to a paid successor (Scrydex). For this
  project, we fetch the mapping **once** via a script and commit the static
  result — no runtime dependency on the API beyond the CDN image hotlink itself.

## Card Source Mapping

- Sets used: `base1` (Base Set, 1999, 102 cards), `base2` (Jungle, 1999, 64
  cards), `base3` (Fossil, 1999, 62 cards) — covers 150/151 Gen-1 Pokémon.
- `#151 Mew` is not in any Base/Jungle/Fossil card; use `basep-8` (Wizards
  Black Star Promos, 1999) — same era, same visual style.
- Where a Pokédex number appears in multiple sets/cards (34 cases, e.g.
  Pikachu in both base1 and base2), prefer the lowest-priority set in this
  order: `base1` > `base2` > `base3` > `basep`. Within the same set, prefer the
  lowest card number.
- Mapping is generated once by `scripts/fetch-tcg-cards.mjs` (queries
  `api.pokemontcg.io/v2/cards?q=set.id:base1` etc., resolves conflicts per the
  rule above, writes `lib/tcg-cards.ts`). The script is run manually during
  implementation; its output is committed. No build-time or request-time
  fetch.

## Data Layer

New file `lib/tcg-cards.ts`:
```ts
// Pokédex # → Pokémon TCG API card image URL (1999 Base/Jungle/Fossil, Mew from
// Wizards Black Star Promos). Generated by scripts/fetch-tcg-cards.mjs.
export const TCG_CARD_IMAGE: Record<number, string> = {
  1: 'https://images.pokemontcg.io/base1/44_hires.png',
  // … all 151 entries
};
```

## Component Changes

### `components/HoloCard.tsx`

- New optional prop `tcgImageUrl?: string`.
- When present, render the card content as:
  - A full-bleed `<img src={tcgImageUrl}>` with `object-fit: contain` filling
    the card area (the real card already shows name/HP/type/attacks/art — no
    overlay needed for those).
  - `onError` on this image falls back to the existing sprite-frame layout
    (current header/sprite-box/type-badge/stat-grid) for that single card, so
    a CDN hiccup never breaks the page.
  - Below the card image, a compact footer strip (new, replaces the old large
    stat `<dl>` grid when `tcgImageUrl` is used): a single-row `ATK / DEF /
    SPD` mini-stat line using our real PokeAPI stats, then the existing
    `footer` prop slot (Add/Remove button) underneath.
- When `tcgImageUrl` is absent, behavior is unchanged (existing sprite-frame
  layout) — used as the error-fallback path and for any caller that doesn't
  pass the prop.
- The holo effect layers (`.holo-card__shine/__sparkle/__glare`) are unchanged;
  they already sit above the content via z-index and blend modes, independent
  of what's in the background.

### Callers

- `components/HomePokedexPreview.tsx` — `toSummary()`'s caller passes
  `tcgImageUrl={TCG_CARD_IMAGE[p.id]}` to `<HoloCard>`.
- `components/PokemonCard.tsx` — same, passes `TCG_CARD_IMAGE[pokemon.id]`.
- `components/PokedexGrid.tsx` — same, for all 151 entries.

## Visual Result

- Card face: real 1999 TCG card art (yellow Base Set border, original name/HP/
  attack text, energy symbols) — pixel-accurate to the reference site.
- Holo/Cosmos shimmer effects layer on top exactly as before (already verified
  working).
- A thin footer strip below the card shows our own ATK/DEF/SPD numbers (so the
  "real API data" requirement stays visible) + the Add/Remove control.

## Out of Scope

- No changes to the rarity-tier logic (Normal/Holo/Cosmos) — unaffected by the
  background swap.
- No battle-screen usage (HoloCard isn't used there).
- No live TCG API calls at runtime — mapping is static/committed.
